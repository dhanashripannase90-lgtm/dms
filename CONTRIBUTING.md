# Contributing to Document Management System

Thank you for your interest in contributing to the DMS project! To ensure a smooth collaboration, please follow these guidelines.

## 🛠️ Development Workflow

1. **Setup**: Follow the instructions in [README.md](./README.md) to set up your local development environment.
2. **Branching**: Create a feature branch for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Coding Standards**:
   - Follow standard Java/Spring Boot practices for the backend.
   - Use functional components and hooks for React.
   - Keep styles consistent with the existing Tailwind/Bootstrap configuration.
4. **Testing**: Ensure your changes do not break existing functionality. Run backend tests using `mvn test`.

## 📝 Pull Request Process

1. Update the documentation if your changes introduce new features or change existing ones.
2. Provide a clear description of the problem you're solving or the feature you're adding.
3. Ensure your code is clean and well-commented where necessary.

## 🏗️ Architecture Notes

- **Backend**: Spring Boot 3.4.x, REST API, Spring Security (JWT).
- **Frontend**: React 19, Axios for API calls, Tailwind CSS for styling.
- **Database**: JPA/Hibernate with MySQL.

## 🆘 Questions?

If you have any questions or need clarification, please reach out to the project maintainer.
